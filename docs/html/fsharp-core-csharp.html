Language posture

Keep the compiler core brutally F#.

Do not design the core around C# compatibility. F# owns the domain model, projection compiler, and evidence logic. C# receives generated adapters at the edge.

Policy

F# owns the design. C# consumes a transcription.

F# Domain / Projection Core -> DomainIR -> generated C# adapter -> OneOf or sealed-record unions -> validators + codecs + fixtures Never: weaken the F# core to make C# authoring convenient

F# Core

Records, DUs, Option, Result, pattern matching, pure modules, constrained values.

C# Adapter

Generated records, OneOf wrappers, sealed record hierarchies, validators, codecs, match helpers.

Golden Tests

F# value to JSON fixture to C# decode/validate/encode back to the same JSON.
