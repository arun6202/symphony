MVP pivot

SQLite is the miniature. Oracle is the production repeat.

The MVP should prove one unified compiler path cheaply with SQLite, then run the same architecture through Oracle end-to-end: harvest, ProjectionSpec, TSV, DuckDB, ETL, Elasticsearch, validation, docs, run ledger, and polyglot contracts.

The product claim

Given a source system or a domain model, generate a governed search/read model with executable evidence and portable client models.

SQLite Miniature

Northwind proves the entire path with low operational cost and fast iteration.

Oracle End-To-End

Oracle repeats the same path with data dictionary harvest, SCN extract, type fidelity, and run evidence.

Polyglot Output

DomainIR generates TypeScript, JavaScript, Kotlin, Swift, and C# OneOf/sealed-record adapters with validators and codecs.

MVP loop: harvest -> compile -> extract -> stage as TSV -> load into DuckDB -> project documents -> write Elasticsearch bulk -> validate -> document -> export client models
