Architecture

SourceCatalog and DomainIR converge into ProjectionSpec.

The core system should describe source truth, domain truth, projection grain, transforms, constraints, lossiness, lineage, quality, and target fields. Every runtime or document output should be a fold over that model.

Boundary Rule

Pure core owns decisions. Shells own effects. If a module opens network, file, or database APIs, it should not also invent projection semantics.

Pure Core

  • SourceCatalog, DomainIR, and ProjectionSpec.
  • Column witnesses, field handles, and expression algebra.
  • Lineage, constraint, lossiness, and enforcement models.
  • Folds to mapping, TSV contracts, metadata, tests, docs, polyglot models, and contracts.

Imperative Shell

  • Oracle, SQLite, TSV, DuckDB, and Kafka/Debezium I/O.
  • Elasticsearch validation and bulk loading.
  • OpenMetadata API push and file emission.
  • API endpoints, frontend delivery, and run logging.

Integrated Flow

Each step should leave a machine-readable artifact and a human-readable explanation.

1
Choose ingress
Brownfield source harvest or greenfield F# domain model.
2
Build common truth
SourceCatalog and DomainIR converge into ProjectionSpec.
3
Extract and stage
SQLite or Oracle emits typed TSV, then DuckDB loads and reconciles.
4
Compile outputs
Elasticsearch mapping, bulk, OpenMetadata, OKF, quality, contracts, docs, and polyglot models come from the same spec.
5
Validate and operate
Versioned index, strict mapping checks, run ledger, alias swap, Helios query surface, and spec diff review.
