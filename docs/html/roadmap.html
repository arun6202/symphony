Phased build

SQLite miniature first, Oracle repeat second, CDC later.

The roadmap now centers on one compiler path: SourceCatalog/DomainIR to ProjectionSpec, TSV to DuckDB staging, ETL to Elasticsearch, run ledger, docs, and polyglot contracts.

Decision Lens

Every phase should produce test evidence, documentation, and one stronger handoff artifact.

Phase 0 - Documentation and shared understanding

Expanded Markdown and HTML docs. New collaborators can orient quickly.

Phase 1 - Reframe SQLite as miniature

Add SourceCatalog and ProjectionSpec drafts around the existing Northwind slice.

Phase 2 - TSV and DuckDB staging

SQLite to typed TSV, TSV manifest/checksums, DuckDB load, reconciliation hooks.

Phase 3 - Closed constraint model

Structured CHECK subset, refined witnesses, constraint coverage, unsupported degradation.

Phase 4 - Streaming projection and bulk

Reader to parser to projector to chunk writer. Bounded memory and checkpoint-ready manifests.

Phase 5 - Oracle harvest and TSV batch

Dictionary harvest, type fidelity, SCN extract, Oracle TSV, DuckDB staging, alias lifecycle.

Phase 6 - Polyglot domain export

DomainIR to TypeScript and C# adapters first, with shared fixtures and codecs.

Phase 7 - Helios query library hardening

Result-based lowering, stronger phantom APIs, golden tests, ES validation.

Phase 8 - Semantic catalog integration

Symphony emits technical seeds; humans enrich; Helios consumes catalog-driven schema.

Phase 9 - CDC prototype

Debezium feasibility, Propulsion.Net projector spike, version guards, tombstones, convergence tests.

Phase 10 - Governance and productization

Spec diff review, owners, PII tags, quality dashboard, run history, operator runbooks.

See ../md/11-roadmap.md for detailed phase notes.
