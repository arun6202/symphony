Frontier features

Build the product around trust, not just generation.

The platform should help humans design projections, review diffs, understand field trust, improve search, replay safely, and hand off work to other engineers or agents.

Most important upgrade

Introduce a first-class ProjectionSpec that can be produced from database-first harvesting or functional DDD declarations.

Projection Workbench

Review source tables, document grain, transforms, lineage, mapping preview, and generated tests in one place.

Evidence Graph

Connect fields to spec hash, source columns, mapping paths, tests, validation runs, and catalog entities.

Run Ledger

Make every extract/load/validate run durable, comparable, auditable, and linkable from docs.

Query Playground

Show user query, typed predicate, lowered ES JSON, validation result, response sample, and field explanations.

Search Quality Loop

Track top queries, zero-result queries, slow queries, relevance judgments, synonyms, and analyzer experiments.

Replay Simulator

Simulate duplicate events, out-of-order events, parent changes, deletes, and batch-vs-stream convergence.

SOTA Elevation Levels

Each level has an exit gate. That is what keeps the architecture from becoming only a vision.

Level 1 - Deterministic artifacts

Stable hashes, deterministic IDs, artifact tests, structured validation output.

Level 2 - Evidence-backed quality

Generated quality suite, coverage dials, ES smoke queries, OpenMetadata validation.

Level 3 - Dual ingress

Database-first SourceCatalog and domain-first DomainProjection converge into ProjectionSpec.

Level 4 - Operational search lifecycle

Versioned indices, alias swap, rollback, reindex planning, run ledger.

Level 5 - Helios product surface

Query playground, semantic catalog browser, field trust page, relevance loop.

Level 6 - CDC convergence

Capture, projector, BulkOp algebra, tombstones, version guards, batch-vs-stream reconciliation.
