Tests and evidence

Make every claim executable.

The docs should not only explain the platform. They should point to tests, generated artifacts, validation output, run ledgers, and acceptance gates.

Core Gate

No governed serving field should ship with Opaque lineage, unapproved lossiness, or unvalidated mapping behavior.

Symphony Artifact Tests

  • Mapping JSON parses and uses dynamic strict.
  • All spec fields exist in mapping.
  • Bulk chunks contain balanced action/source pairs.
  • Manifest count equals emitted documents.
  • OpenMetadata bundle contains expected entity types.

Helios Query Tests

  • Golden JSON for every lowering rule.
  • Negative compilation tests for illegal operators.
  • Property tests for bool flattening and codec round-trip.
  • Elasticsearch `_validate/query` integration checks.
  • Mutation tests around lowering.

Lineage

Exact, Declared, and Opaque are coverage dials. Gates decide where each grade is acceptable.

Constraints

Prevented, Inherited, and Detected classify what is proven, what is carried, and what is reconciled.

Lossiness

Precision, timezone, blob, and text-analysis loss should be explicit and reviewable.
