# Canonflow Brand

Canonflow is the umbrella name for the platform.

```text
Canonflow
  Symphony -> compiler core
  Helios   -> workbench, query, catalog, and product surface
```

## Why Canonflow Works

Canon means:

- canonical truth
- source of record
- rules
- contracts
- accepted standard

Flow means:

- pipeline
- movement
- lineage
- projection
- delivery

Together:

> Canonflow compiles canonical truth into governed data products.

## Preferred Tagline

```text
Compile source truth into governed data products.
```

Alternate taglines:

- Canonical truth, governed flow.
- From source truth to governed search.
- The data product compiler for trusted read models.
- Where domain truth becomes production search.
- Governed read models from source and domain truth.

## Product Positioning

Canonflow is not only an ETL tool, catalog, ORM, or query DSL.

It is:

> A type-driven data product compiler for trusted read models, governed search, lineage, quality, contracts, docs, and polyglot consumer models.

## Naming Model

Use the names like this:

- **Canonflow** - whole platform and product.
- **Symphony** - F# compiler core, projection engine, evidence generator.
- **Helios** - human/product surface: query workbench, semantic catalog, field trust, search quality.

Examples:

```text
Canonflow uses Symphony to compile ProjectionSpec artifacts.
Canonflow uses Helios to let users query, inspect, and evolve generated read models.
```

## Message Map

For architects:

> Canonflow turns database-first and domain-first truth into governed, testable, searchable read models with evidence attached.

For F# developers:

> Canonflow keeps the core brutally F#: records, DUs, constraints, Result, pattern matching, pure folds.

For enterprise data teams:

> Canonflow harvests source systems, generates lineage and contracts, validates Elasticsearch surfaces, and records run evidence.

For product teams:

> Canonflow creates searchable, explainable, governed data products that humans can trust and improve.

For polyglot consumers:

> Canonflow exports safe models, validators, codecs, and selected pure rules to TypeScript, JavaScript, Kotlin, Swift, and C# adapters.

## Visual Language

The brand should feel:

- precise
- trustworthy
- technical
- calm
- evidence-oriented
- practical

Avoid:

- mystical language
- vague AI language
- generic "data platform" wording
- overpromising CDC or real-time magic before batch correctness is proven

## Canonflow Architecture Sentence

```text
Canonflow: SourceCatalog or DomainIR -> ProjectionSpec -> TSV/DuckDB/ETL -> Elasticsearch -> evidence, docs, catalog, contracts, and polyglot models.
```
